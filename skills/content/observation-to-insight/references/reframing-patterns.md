# Reframing Patterns

Systematische Kategorie-Verschiebungen. Jedes Pattern = eine Denkbewegung, die Beobachtungen in Insights transformiert.

## Pattern 1: Genre-Verwechslung

**Move:** Was als X gelesen wird, gehört eigentlich in Genre Y.

| Surface Genre | Actual Genre | Example Domain |
|---|---|---|
| Ideologie | Entertainment | Political content |
| Meinung | Bedürfnis-Signal | Consumer behavior |
| Fehler | Feature | Platform design |
| Trend | Symptom | Cultural shifts |
| Authentizität | Performance | Personal branding |
| Information | Positioning | Thought leadership |
| Debatte | Statusspiel | Public discourse |

**Test:** Erklärt das neue Genre das Verhalten besser als das alte?

---

## Pattern 2: Optimierungs-Ziel entlarven

**Move:** System optimiert nicht für X (behauptet), sondern für Y (tatsächlich).

| Behauptetes Ziel | Tatsächliches Ziel |
|---|---|
| User experience | Engagement/Retention |
| Sicherheit | Liability-Reduktion |
| Qualität | Skalierbarkeit |
| Transparenz | Kontrolle |
| Community | Datenextraktion |
| Innovation | Investor-Narrativ |

**Test:** Welches Ziel erklärt die Design-Entscheidungen, die "keinen Sinn ergeben"?

---

## Pattern 3: Ebenen-Verschiebung

**Move:** Problem sitzt nicht auf Ebene X (wo es diskutiert wird), sondern auf Ebene Y.

| Diskutierte Ebene | Eigentliche Ebene |
|---|---|
| Individuum | Institution |
| Moral | Anreizstruktur |
| Kompetenz | Zugang |
| Kultur | Ökonomie |
| Intention | Infrastruktur |
| Inhalt | Distribution |

**Test:** Löst Intervention auf der neuen Ebene das Problem nachhaltiger?

---

## Pattern 4: Zeitstruktur umkehren

**Move:** Was als Ursache gilt, ist eigentlich Folge (oder umgekehrt).

Examples:
- "Vertrauen ermöglicht Kooperation" → "Kooperations-Infrastruktur erzeugt Vertrauen"
- "Guter Content führt zu Reichweite" → "Reichweiten-Logik formt Content"
- "Werte prägen Verhalten" → "Verhalten rationalisiert sich als Werte"

**Test:** Stimmt die Chronologie? Was kam wirklich zuerst?

---

## Pattern 5: Akteur-Struktur-Flip

**Move:** Nicht X (Akteur) ist verantwortlich, sondern Y (Struktur) – oder umgekehrt.

Varianten:
- **Struktur → Akteur:** "Das System zwingt niemanden" (agency restoration)
- **Akteur → Struktur:** "Jeder einzelne verhält sich rational, das Ergebnis ist irrational" (composition fallacy)
- **False binary:** Weder Akteur noch Struktur, sondern deren **Schnittstelle** (Protokoll, Norm, Interface)

**Test:** Wer/was hat tatsächlich die Entscheidungsgewalt?

---

## Pattern 6: Negativ-Definition

**Move:** X definiert sich nicht durch was es ist, sondern durch was es ausschließt/verhindert.

Examples:
- "Professionalität" = Ausschluss von Emotionalität
- "Qualitätsjournalismus" = Abgrenzung von Social Media
- "Wissenschaftlichkeit" = Legitimation durch Methode, nicht Ergebnis

**Test:** Was kann innerhalb dieser Kategorie nicht existieren? Das ist die eigentliche Definition.

---

## Pattern 7: Funktion statt Intention

**Move:** Nicht "warum wurde X gemacht?" sondern "welche Funktion erfüllt X im System?"

Unterscheide:
- **Manifeste Funktion:** Erklärter Zweck
- **Latente Funktion:** Tatsächlicher System-Nutzen
- **Dysfunktion:** Unbeabsichtigte Nebeneffekte, die aber stabilisierend wirken

**Test:** Würde das System kollabieren, wenn X wegfiele? Dann ist das die Funktion.

---

## Pattern 8: Knappheits-Inversion

**Move:** Was knapp scheint, ist abundant – und umgekehrt.

| Scheinbar knapp | Tatsächlich knapp |
|---|---|
| Information | Aufmerksamkeit |
| Content | Kuratierung |
| Meinung | Urteilsfähigkeit |
| Optionen | Entscheidungskriterien |
| Zugang | Kontext |
| Daten | Vertrauen |

**Test:** Wofür wird tatsächlich Geld/Zeit/Status bezahlt?

---

## Pattern 9: Normalisierungs-Archäologie

**Move:** Was "normal" ist, war irgendwann Erfindung/Entscheidung.

Fragen:
- Wann wurde X zum Standard?
- Wer profitierte von dieser Standardisierung?
- Welche Alternativen wurden dabei ausgeschlossen?
- Was müsste passieren, damit X wieder als "Entscheidung" sichtbar wird?

**Test:** Kann man einen historischen Moment benennen, an dem X noch nicht selbstverständlich war?

---

## Pattern 10: Paradoxie-Auflösung

**Move:** Scheinbarer Widerspruch ist keiner, wenn man die versteckte Ebene/Variable findet.

Struktur:
1. Benenne das Paradox ("Einerseits X, andererseits nicht-X")
2. Finde die versteckte Variable ("...aber nur unter Bedingung Y")
3. Löse auf ("X und nicht-X adressieren verschiedene Ys")

**Test:** Verschwindet der Widerspruch, wenn man die Variable expliziert?

---

## Kombinations-Regel

Die stärksten Insights kombinieren zwei Patterns:
- Genre-Verwechslung + Optimierungs-Ziel → "Das ist kein Bug, das ist das Feature"
- Ebenen-Verschiebung + Funktion statt Intention → Institutionenkritik ohne Moralisierung
- Knappheits-Inversion + Negativ-Definition → Markt-/Status-Analyse

**Warnung:** Mehr als zwei Patterns = zu komplex für Reel-Format.
